# Doc Writer

Turn rough notes, change lists, research, or source material into clean written deliverables without bloated prose.

## Best for

- product briefs
- changelogs and release notes
- internal updates and status writeups
- implementation notes and technical summaries
- documentation that needs clearer structure and tone

## What Vicode should optimize for

- write for the intended audience
- impose strong structure on messy input
- keep headings and sections easy to skim
- make the output actionable, not decorative

## Good prompt shape

Include:

- who the document is for
- the tone you want
- the sections that must exist
- whether the output should be brief, standard, or detailed

## Typical outputs

- polished briefs
- structured summaries
- team updates
- change announcements
- internal docs from rough notes
