# Vicode 0.2.6

Date: 2026-04-26

## Summary

`0.2.6` is the next Windows beta patch line for tester safety. It hardens the
boundary between Vicode and the OpenAI Codex app so Vicode does not use the
operator's real Codex app database, automation folders, worktrees, sessions, or
skills as Vicode state.

## Major Changes

### Codex app boundary

- Vicode-managed skills no longer export, sync, install, uninstall, or clean
  files inside the operator's real `~/.codex/skills` folder.
- Build Control no longer falls back to the operator's real `~/.codex`
  automation prompts when repo-local lane prompts are missing.
- Repo-local control helpers now require an explicit isolated Codex home or
  explicit non-Codex paths instead of defaulting to the operator's real
  `~/.codex/sqlite`, `~/.codex/automations`, or `~/.codex/worktrees`.

### Beta documentation

- The beta tester docs now call out the separation between Vicode app state and
  Codex app state.
- Public source mirror scripts now include the current beta release notes and
  reviewer guide so source users land on the same release line as installer
  users.

## Packaging Notes

- App version: `0.2.6`
- The Windows installer artifact for this line is `Vicode-Setup-0.2.6.exe`.
- `latest.yml` for this line should point at that same installer artifact.
- Desktop update artifacts continue to publish to the public release-only repo
  `lowluds/vicode-windows-releases`.

## Verification

Required before public handoff:

- `npm test`
- `npm run build`
- `npm run dist:win`
- `npm run smoke:packaged`
- `npm run audit:release`
- `npm run audit:repo-public`
- `npm run audit:public-mirror`
