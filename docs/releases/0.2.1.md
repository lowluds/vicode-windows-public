# Vicode 0.2.1

Date: 2026-04-18

## Summary

`0.2.1` is a release-hardening update for the current Windows-first Vicode
line. It tightens packaged-app validation, improves live-provider failure
diagnostics, and adds first-class direct-chat creation on the collaboration
side without changing the core product stance: local coding workflows remain
the release-defining surface.

## Major Changes

### Collaboration

- Added first-class direct-chat creation from shared contacts.
- Kept collaboration inside the same guest-room room/session model instead of
  creating a separate parallel transport.

### Packaged Windows validation

- Added packaged-app smoke coverage for `release/win-unpacked/Vicode.exe`.
- Added release-output and public-snapshot audit scripts so packaged artifacts
  and current-facing docs can be checked for repo-only files and machine-local
  path leakage before handoff.
- Added installed-app desktop update wiring for the packaged Windows path, with
  runtime status surfaced in `Settings > General`.
- Added a visible titlebar update action so downloaded updates can queue until
  the current run finishes instead of interrupting an active run.

### Diagnostics and certification

- Live-provider certification failures now include the last raw provider events
  in Playwright output, which makes release-gate debugging more direct.
- Collaboration verification now includes deterministic direct-chat coverage in
  storage, service, and Electron UI tests.

### Fixes

- The packaged Windows path now uses a real runtime updater instead of only
  generating release metadata files without app-side wiring.

## Packaging Notes

- App version: `0.2.1`
- Current external beta delivery path for this line is the packaged Windows
  installer and unpacked desktop app.
- The Windows installer artifact for this line is `Vicode-Setup-0.2.1.exe`.
- `latest.yml` for this line now points at the same installer artifact name.
- Installed Windows builds now check GitHub Releases for updates on launch and
  show update state in both `Settings > General` and the titlebar.
- Downloaded desktop updates can be queued from the titlebar while a run is
  active, then installed automatically once that run finishes.
- The desktop update feed now targets the public release-only repo
  `lowluds/vicode-windows-releases`.
- Public beta source snapshots should publish through the curated mirror repo
  `lowluds/vicode-windows-public`, not by exposing the internal working repo
  history directly.
- `npm pack` / `npm publish` remain a separate audit lane and are not yet the
  supported end-user install path for this release line.
