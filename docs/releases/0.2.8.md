# Vicode 0.2.8

Date: 2026-06-05

## Summary

`0.2.8` is the current Windows beta readiness line. It carries the recent composer
mode verification, clearer execution warning copy, local Ollama API beta
positioning, OpenAI-compatible Custom API positioning, and release-readiness
documentation refresh.

This version does not claim upstream `ollama launch vicode` support yet.

## Improved

- Composer edit-mode controls now use clearer language for direct edits,
  proposed changes, isolated worktrees, and full-access warnings.
- Direct edits, proposed changes, and isolated worktree behavior now have
  front-facing Playwright coverage so the thread evidence matches the selected
  edit mode.
- Ollama beta copy now treats the local Ollama API as the supported Ollama path.
- OpenAI-compatible beta copy now points testers to `Custom API` saved API keys
  rather than first-class OpenAI CLI setup.

## Known Beta Caveats

- Vicode is not yet listed by upstream Ollama as an `ollama launch` integration.
- OpenAI keys are supported through `Custom API` as an OpenAI-compatible
  provider. First-class OpenAI CLI setup is retired for this beta route.
- Ollama API keys are retired for this beta route. Use the local Ollama runtime,
  or add OpenAI-compatible keys through `Custom API`.
- The Windows installer signer status is `NotSigned` for this candidate and
  must be caveated in beta handoff copy.
- The rebuilt artifact passed fresh-install proof and `0.2.7` to `0.2.8`
  upgrade proof. A prior pre-merge rapid reinstall attempt reproduced one
  installer crash with exit `-1073741819`; watch for that pattern during final
  human testing.
- Draft release asset URLs are not the public tester links. The draft assets now
  match the rebuilt candidate, but fill the handoff copy only after the release
  is published or after an explicit private-invite decision.
