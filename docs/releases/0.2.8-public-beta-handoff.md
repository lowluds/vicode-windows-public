## Reddit-Style Post Draft

Title:

```text
Looking for a few Windows beta testers for Vicode, a small Ollama-first coding app
```

Body:

```text
I am opening a limited public beta for Vicode on Windows.

Vicode is a desktop coding shell: open a local project, start a thread, connect a provider, ask for small code changes, and review what changed. It is intentionally narrower than a full IDE or autonomous builder.

Current beta scope:
- Windows installer: [RELEASE ASSET LINK]
- Recommended provider: local Ollama
- API-key/cloud option: Custom API with an OpenAI-compatible /v1 base URL and model
- OpenAI keys are one Custom API option; other compatible providers can be tested through the same path
- OpenAI keys should be added through Custom API, not through Codex/OpenAI CLI setup

Good first tests:
- install and launch the app
- open a small local project
- ask it to explain a file
- ask it to make a small docs or UI change
- ask it to fix a tiny bug
- close and reopen the app to see whether state restores
- uninstall when done

Known limits:
- the installer is currently unsigned, so Windows may warn before install
- this is a limited beta, not a production-ready autonomous builder
- local Ollama model writing is still model-dependent, so broad local-model file-writing claims are not the promise yet
- first-class OpenAI CLI, Gemini, Qwen, and Kimi CLI setup are retired from this beta route
- hosted/cloud Ollama-specific key setup is not a separate beta route; use Custom API only if the service exposes an OpenAI-compatible /v1 endpoint

Feedback I need most:
- did install or launch fail?
- was provider setup clear?
- did it understand your project files?
- did edits land where you expected?
- did approval/warning copy make sense?
- did relaunch or uninstall behave strangely?

Please file issues here:
https://github.com/lowluds/vicode-windows-public/issues/new?template=beta-feedback.yml
```

## Feedback Questions

- What Windows version did you test on?
- Did the installer complete? If not, what warning or error appeared?
- Did first launch reach the normal app shell?
- Which provider did you use: local Ollama or Custom API?
- If you used Custom API, which OpenAI-compatible base URL and model family did
  you test?
- Did provider setup need anything that was not explained?
- What project type did you open?
- Did Vicode correctly explain a real file from that project?
- Did it make the requested small edit or bug fix?
- Were tool approvals, warnings, and changed-file evidence understandable?
- Did the thread/project restore after relaunch?
- Did uninstall remove the app cleanly?
