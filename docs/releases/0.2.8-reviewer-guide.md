# Vicode 0.2.8 Reviewer Guide

Date: 2026-06-04

## Focus

Review `0.2.8` as a Windows beta readiness candidate. The important checks are
the composer edit-mode surfaces, honest Ollama/OpenAI-compatible API-key beta
positioning, exact candidate packaging proof, the automation-backed 12-check
walkthrough proxy, and the remaining human/outside-tester judgement pass.

Do not accept this line as public-handoff ready from `0.2.7` evidence alone,
from automated gates alone, or from pre-merge package hashes. The post-merge
`0.2.8` installer, packaged app, installed app, public-source audits, local
Ollama setup proof, and OpenAI-compatible Custom API proof were refreshed on
2026-06-05 for the local-only provider-scope candidate worktree. Public posting
still needs the human/outside-tester judgement pass below or an explicit scoped
exception, plus a publish or private-invite link decision. The packaged app
code boundary is committed at `db56682`; later branch commits record draft
asset parity, documentation/evidence cleanup, the human judgement log, public
issue intake repair, and public mirror refresh.

## Primary Checks

Review and, before public posting, run the full 12-check manual walkthrough from
[public-beta-verification-goal-0.2.8.md](../engineering/public-beta-verification-goal-0.2.8.md)
and record the result in
[0.2.8-human-judgement-log.md](./0.2.8-human-judgement-log.md):

1. Fresh install on Windows.
2. First launch works without dev tools or repo context.
3. Open a normal local project folder.
4. Connect the supported provider path, currently Ollama or Custom API with an OpenAI-compatible API key.
5. Ask Vicode to explain a file.
6. Ask Vicode to make a small docs or UI change.
7. Ask Vicode to fix a tiny bug.
8. Confirm tool approvals and warnings make sense.
9. Relaunch and confirm thread/project state persists.
10. Install over the previous beta if upgrade users are expected.
11. Uninstall and confirm no confusing leftover behavior.
12. Confirm tester docs point to the installer, not source zip or npm.

Additional reviewer checks:

- Confirm the composer edit-mode controls expose direct edits, proposed changes,
  and isolated worktree behavior with clear warning copy.
- Confirm Ollama copy points to the local Ollama runtime/API, not an Ollama API
  key.
- Confirm OpenAI copy points testers to `Custom API` / OpenAI-compatible
  API-key setup, not first-class OpenAI CLI setup.
- Confirm the public beta handoff copy in
  [0.2.8-public-beta-handoff.md](./0.2.8-public-beta-handoff.md) has real
  release and issue links before posting.
- Confirm Vicode does not write provider-native skills or automation state into
  another provider application's private app-state folders.
