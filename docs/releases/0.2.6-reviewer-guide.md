# Vicode 0.2.6 Reviewer Guide

This guide is for someone reviewing Vicode for the first time on Windows.

It explains:

- what Vicode is
- how beta testers should install it
- how to run it locally from source when needed
- how to build the Windows installer and app
- what changed in the `0.2.6` safety-hardening line

## What Vicode Is

Vicode is a Windows-first Electron desktop app for AI-assisted coding work.

It is intentionally smaller than Codex or full IDE shells. The current release
line focuses on:

- local project and thread workflows
- trustworthy provider execution
- a calm transcript-based desktop UI
- packaged Windows beta delivery

## Current Release Posture

For `0.2.6`, the current external beta path is the packaged Windows app:

- `release/Vicode-Setup-0.2.6.exe`
- `release/win-unpacked/Vicode.exe`

This release replaces the earlier `0.2.5` beta build. The main user-facing
correction is safety hardening around the OpenAI Codex app boundary:

- Vicode app state and Codex app state are treated as separate applications.
- Vicode does not write, delete, sync, install, uninstall, or clean files inside
  the operator's real `~/.codex/skills` folder.
- Build Control and repo-local automation helpers must use isolated Codex homes
  or explicit non-Codex paths when they need persistent Codex-like state.

Installed Windows builds continue to check GitHub Releases for desktop updates
on launch, show update state in `Settings > General`, and surface a visible
titlebar update action when a new build is ready.

## Beta Tester Quick Start

For a first-time beta tester, use this path:

1. Download the installer from `lowluds/vicode-windows-releases`.
2. Install and launch Vicode on Windows.
3. Open a project folder to begin.
4. Connect `OpenAI / Codex CLI`, `Gemini CLI`, or `Ollama`.
5. Use the titlebar update action or `Settings > General` when an update is ready.
6. File public bug reports through GitHub Issues.

## Requirements

Recommended local baseline:

- Windows 11
- Node.js `24.x`
- npm `10+`

Install dependencies:

```bash
npm install
```

If Electron native dependencies drift after install or after an Electron
upgrade, repair them with:

```bash
npm run native:prepare:electron
```

## Run The App Locally

Start the desktop app in development mode:

```bash
npm run dev
```

## Test The Current Desktop Build

The focused verification for `0.2.6` is:

```bash
npm test
npm run build
npm run dist:win
npm run smoke:packaged
npm run audit:release
npm run audit:repo-public
npm run audit:public-mirror
```

The broader deterministic sweep remains:

```bash
npm run verify:desktop
```

## Build The Windows Installer And Desktop App

Create the packaged Windows artifacts:

```bash
npm run dist:win
```

Then validate the packaged output:

```bash
npm run smoke:packaged
npm run audit:release
npm run audit:repo-public
npm run audit:public-mirror
```

Publish the Windows update channel only when the candidate is backed by a tag
or draft release in `lowluds/vicode-windows-releases`:

```bash
npm run release:win
```

## Current Beta Reality

Current repo-local beta truth:

- packaged Windows delivery remains the supported tester path
- clicking the downloaded-update action restarts the app immediately, even when
  a run is currently active
- Vicode should not mutate the operator's Codex app files under the real
  `~/.codex`
- GitHub Issues is the supported public path for bug reports
- npm packaging remains an audit lane, not the supported public install route
